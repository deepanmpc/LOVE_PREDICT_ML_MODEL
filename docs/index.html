How It Works

Answer 30 Questions

The web UI walks you through 30 behavioral and psychological questions about both individuals, one at a time.

02

Feature Engineering

Responses are scaled using a fitted StandardScaler and fed into the trained ML model via a Flask endpoint.

XGBoost or Random Forest classifier runs inference on the 30-dimensional feature vector in milliseconds.

Compatibility Result

The model returns a class — Just Friends, Could Work Out, or Strong Match — displayed with animation.

Output

The Three Classes

Class 0

Just Friends

Compatibility signals suggest the relationship is better suited to a platonic dynamic. Differences outweigh shared emotional alignment.

Class 1

Could Work Out

Mixed signals — some strong compatibility markers exist alongside areas of friction. Growth and communication could bridge the gaps.

Class 2

Strong Match

High compatibility across behavioral, emotional, and psychological features. The model sees strong alignment across most dimensions.

Integration

The API

The Flask backend exposes a simple /predict endpoint. POST your 30 features, get a compatibility class back. 

# POST to the prediction endpoint
curl -X POST http://localhost:5000/predict \
     -H "Content-Type: application/json" \
     -d '{
       "communication_frequency": 8,
       "trust_score": 9,
       "conflict_resolution": 7,
       "emotional_support": 8,
       ...
       "reaction_to_praise": 2
     }'
import requests

features = {
    "communication_frequency": 8,
    "trust_score": 9,
    "conflict_resolution": 7,
    # ... all 30 features
}

response = requests.post(
    "http://localhost:5000/predict",
    json=features
)
print(response.json())
# Class 0 — Just Friends
{ "prediction": 0 }

# Class 1 — Could Work Out

# Class 2 — Strong Match
{ "prediction": 2 }
